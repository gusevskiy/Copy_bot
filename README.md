# Copy_bot
Бот для удобства отслеживания контента.
Пересылает сообщения из одной или нескольких групп, на которые подписан владелец, в одну или несколько других групп.

Пересылает всё: текст, картинки, голосовые и видео (в том числе альбомы — группируются в рамках одного сообщения).

Для работы бота на сервере нужно:

[Docker](https://docs.docker.com/engine/install/ubuntu/)

[Client pyrogram](https://docs.pyrogram.org/intro/quickstart) — юзербот, который читает сообщения из донор-чатов (нужен `.session`-файл авторизованного аккаунта)

[Bot telegram](https://telegram.me/BotFather) — обычный бот, через который контент публикуется в чаты-получатели

```bash
git clone https://github.com/gusevskiy/Copy_bot
cd Copy_bot
```

Рядом с `docker-compose.yaml` нужно положить:
- файл `.env` с переменными окружения (см. формат ниже)
- `.session`-файл юзербота (имя должно совпадать с `SESSION` из `.env`), путь к нему прописан в `docker-compose.yaml` в секции `volumes`

```bash
docker compose up --build -d
```

## Формат `.env`

В `DONOR` — список ID групп, из которых нужно получать контент.
В `RECIPIENT` — список ID групп, в которые нужно пересылать контент.
Получение и доставка сопоставляются по индексу в списке: `DONOR[i]` → `RECIPIENT[i]`.

Если нужно пересылать несколько доноров в одну и ту же группу — просто повторите ID получателя нужное количество раз, по числу доноров:
```
DONOR=111111, 222222, 333333, 444444
RECIPIENT=555555, 555555, 666666, 666666
```
Сообщения из `111111` и `222222` пойдут в `555555`, а из `333333` и `444444` — в `666666`.

Полный пример `.env`:
```
DONOR=111111, 222222
RECIPIENT=333333, 333333
SESSION=name_session
# Токен обычного бота (aiogram), через который идёт публикация
TOKEN=TOKENTOKENTOKENTOKENTOKENTOKENTOKENTOKENTOKEN
```
